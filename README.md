## ☀️ About

Transformer-based models have made token sequences a universal interface for modern AI systems.
In the era of AI agents, tokens no longer represent only task inputs — they also capture
observations, retrieved evidence, and memory records accumulated during multi-step execution.
**Token compression** becomes a critical active context optimization problem:
how to preserve task-relevant information with fewer active tokens under limited context
window and computation budgets.

This repository accompanies our survey on
**"A Survey of Token Compression in the Era of AI Agents: From Perception to Semantics"**
by Fengxi Zhang, Zhengxue Cheng, and Li Song.

Our **agent-centric taxonomy** divides token compression into two main categories:

- **Perception Compression** — reduces input-side perceptual tokens (text, image, video, audio)
  by exploiting redundancy in raw inputs or perceptual representations.
- **Semantic Compression** — reduces workflow-context tokens (observations, retrieval, memory)
  accumulated during agent execution by preserving decision-relevant information.

Feel free to browse the paper tables, open issues, or contribute entries.

## 👁️ Perception Compression

Perception compression exploits the intrinsic redundancy of raw perceptual inputs to condense them into compact token representations. This section covers **199 papers** across four modalities, categorized into four paradigms: **Transformation**, **Token Selection**, **Token Aggregation**, and **Token Resampling**.

### Text Token Compression

Text token compression exploits the statistical redundancy in natural language to reduce prompt and context length fed into LLMs, going beyond vocabulary-level tokenization to remove context-level redundancy.

**Transformation** — reduces token length via structural operations (convolution, pooling, pixel unshuffle, etc.).

### Memory Token Compression

Memory token compression manages historical traces and experience records in long-horizon agent workflows, organized into memory formation, retrieval, and management stages.

**Formation** — transforms complete histories into compact and reusable memory representations.
